# 为YNU快递代拿做贡献
首先，感谢您抽出时间为我们的小程序做出贡献！是像你这样的人帮助我们才会开花结果。遵循这些指导原则有助于传达这样的信息：您尊重开发人员管理和开发这个开源项目的时间。作为回报，他们应该在解决您的问题或评估补丁程序和功能时回报这种尊重。
## 说明
代码库中的所有更改都应该源于GitHub问题，并利用现有的GitHub功能。目标是吸引贡献者并允许他们同时对代码和文档做出贡献。



创建本文档中的指导原则是为了帮助新的和现有的参与者理解流程工作流，并在提交请求之前与项目规则保持一致。它解释了参与者应该如何进行格式化代码、测试修复和提交补丁程序等操作。
## 核心思想

我们尽量避免添加配置和标志。此工具的目的是为开始使用的用户提供最佳体验，这始终是我们的首要任务。这意味着有时我们会牺牲额外的功能（比如服务器渲染），因为很难用一种不需要任何配置的方式来解决它。



我们更喜欢约定、启发式或交互性而不是配置。

## 我们可以做什么
+ 如果要报告文档中的错误或问题，请提交提供问题详细信息的问题。如果您认为该问题需要优先关注，请对该问题发表意见并通知团队。
+ 若要建议新功能，请提交新功能请求问题。描述预期的特性，并与团队和社区讨论设计和实现。一旦团队认为计划看起来不错，就按照贡献的代码继续执行并实现它。
+ 要为现有的未解决问题实现功能或错误修复，请遵循贡献的代码。如果你需要一个特定问题的更多背景信息，可以对这个问题发表评论，让人们知道。

## 如何进行贡献
+ 如果你对小程序有改进，把你的请求发给我们！对于刚刚起步的用户，请参阅GitHub工作流程。确保在请求的注释和更新中引用changelog.md。
+ 我们团队成员将审查您的拉取请求，一旦它被接受，与之关联的项目的状态将更改为Reviewer approved。这意味着我们正在将您的请求提交到内部存储库。在内部提交更改后，您的请求将在GitHub上自动合并。
+ 可以直接联系我们的小队成员进行讨论交流。
